# タイムカードアプリ　セットアップガイド

このガイドに沿って進めると、スマホで使えるタイムカードアプリが公開できます。
所要時間の目安：30〜45分

---

## STEP 1：Node.js をインストールする

アプリをビルド（完成形に変換）するために必要なソフトです。

1. https://nodejs.org にアクセス
2. 「LTS（推奨版）」のボタンをクリックしてダウンロード
3. ダウンロードしたファイルを開いてインストール（全て「次へ」で OK）

---

## STEP 2：Supabase のプロジェクトを作る

データを保存するサービスです。無料で使えます。

1. https://supabase.com にアクセスして「Start for free」をクリック
2. GitHub アカウントでサインアップ（GitHub がなければメールアドレスで登録）
3. ダッシュボードで「New Project」をクリック
4. 以下を入力して「Create new project」をクリック
   - Name：timecard（何でも OK）
   - Database Password：自分で決めたパスワード（メモしておく）
   - Region：Northeast Asia（Tokyo）

プロジェクトの作成に1〜2分かかります。

---

## STEP 3：データベースを作る

1. 左メニューの「SQL Editor」をクリック
2. 「New query」をクリック
3. `supabase/schema.sql` ファイルの中身を全てコピーして貼り付ける
4. 右上の「Run」ボタンをクリック
5. 「Success」と表示されれば OK

---

## STEP 4：Google ログインを設定する

### 4-1：Google Cloud でプロジェクトを作る

1. https://console.cloud.google.com にアクセス
2. 上部の「プロジェクトの選択」→「新しいプロジェクト」をクリック
3. プロジェクト名：timecard-app（何でも OK）→「作成」
4. 左メニュー「APIとサービス」→「OAuth 同意画面」
5. 「外部」を選択して「作成」
6. アプリ名に「タイムカード」、メールアドレスを入力して「保存して次へ」
7. スコープは何も変更せず「保存して次へ」
8. 「テストユーザー」で「+ ADD USERS」→自分のメールアドレスを追加→「保存して次へ」

### 4-2：OAuth クライアント ID を作る

1. 左メニュー「APIとサービス」→「認証情報」
2. 「認証情報を作成」→「OAuth クライアント ID」
3. アプリケーションの種類：「ウェブアプリケーション」
4. 名前：timecard（何でも OK）
5. 「承認済みのリダイレクト URI」に以下を追加：
   ```
   https://【SupabaseのプロジェクトURL】/auth/v1/callback
   ```
   ※ Supabase のプロジェクト URL は STEP 5-1 で確認できます
6. 「作成」→ クライアント ID とクライアントシークレットが表示されるのでメモ

### 4-3：Supabase に Google の設定を入力する

1. Supabase ダッシュボードに戻る
2. 左メニュー「Authentication」→「Providers」
3. 「Google」をクリックして有効化
4. 「Client ID（for OAuth）」と「Client Secret（for OAuth）」に 4-2 でメモした値を入力
5. 「Save」をクリック

---

## STEP 5：アプリの設定ファイルを作る

### 5-1：Supabase の URL とキーを確認する

1. Supabase ダッシュボードの左メニュー「Project Settings」→「API」
2. 「Project URL」と「anon / public」のキーをメモ

### 5-2：.env ファイルを作る

`timecard-app` フォルダの中に `.env` という名前のファイルを作成して、以下を入力：

```
VITE_SUPABASE_URL=【Project URLをここに貼る】
VITE_SUPABASE_ANON_KEY=【anon keyをここに貼る】
VITE_INVITE_TOKEN=【自分で決めた招待コードをここに入力（例：timecard2024）】
```

---

## STEP 6：アプリをビルドする

1. `timecard-app` フォルダを開く
2. アドレスバーに `cmd` と入力して Enter（コマンドプロンプトが開く）
3. 以下のコマンドを順番に入力して Enter：

```
npm install
```
（パッケージのインストール。1〜2分かかります）

```
npm run build
```
（アプリのビルド。完了すると `dist` フォルダができます）

---

## STEP 7：Netlify で公開する

1. https://netlify.com にアクセスして「Sign up」→ GitHub などでサインアップ
2. 「Add new site」→「Deploy manually」
3. `timecard-app/dist` フォルダをドラッグ＆ドロップ
4. 数秒でURLが発行されます（例：https://amazing-name-123.netlify.app）

### Netlify に環境変数を設定する

1. サイトの管理画面で「Site configuration」→「Environment variables」
2. 以下の3つを「Add a variable」で追加：

| Key | Value |
|-----|-------|
| VITE_SUPABASE_URL | SupabaseのProject URL |
| VITE_SUPABASE_ANON_KEY | Supabaseのanon key |
| VITE_INVITE_TOKEN | 自分で決めた招待コード |

3. 「Deploys」→「Trigger deploy」→「Deploy site」でビルドし直す

---

## STEP 8：リダイレクト URL を追加する

Google OAuth のリダイレクト先に Netlify の URL を追加します。

1. Google Cloud Console の「認証情報」→ 作成したOAuthクライアントを開く
2. 「承認済みのリダイレクト URI」に以下を追加：
   ```
   https://【NetlifyのURL】/auth/v1/callback
   ```
3. 「保存」

---

## STEP 9：招待 QR コードを作る

1. https://qr.quel.jp などの無料 QR コード作成サービスにアクセス
2. 以下の URL を QR コードにする：
   ```
   https://【NetlifyのURL】/?invite=【STEP5で決めた招待コード】
   ```
   例：`https://amazing-name-123.netlify.app/?invite=timecard2024`
3. QR コードを印刷してスタッフに配布

---

## 完成！

QR コードをスタッフが読み取ると、Googleアカウントでログインして使い始めることができます。

2回目以降はQRコードなしで、ブックマークや直接URLからアクセスできます。

---
